# Retrospective (Eigen reflectie)
Met deze retrospective geef ik een eigen reflectie via de **STARRT** Formulieren die op de knowledgebase te vinden zijn.

## Situatie

### Wat was de situatie?

Begin blok 3, kreeg ik te horen dat we een game moesten maken, dat verbaasde me wel een beetje doorwat ik software engineering doe en geen game development. Ik dacht eerst dat we de game engine zelf moesten maken, omdat we met lit en express moesten oefenen. Dit zou ik ook geen probleem vinden aangezien ik al een keertje met react heb gewerkt wat heel erg op lit lijkt. Maar toen kregen we te horen dat we de game engine al kregen. Ik moest wel een beetje wennen aan de code base, dus heb ik de videos van Lennard gekeken + het project zelf door bekijken. Sprint 1 moest iedereen dus nog wennen aan alles, het mergen, de code base + ook nog de storyline bedenken. Dit is allemaal gelukt tijdens sprint 1 maar nog niet vlekkeloos. Tijdens sprint 2 (deze sprint) heb ik me vooral gefocust op mijn kamer meer functionaliteit geven + de samenhang tussen de andere kamers.

### Wanneer speelde de situatie zich af?

5 Februari tot 5 april.

### Wat speelde er?

Ik was bezig met mijn kamer, de persoon daarin + de laptop in de kamer.

### Wie waren er bij betrokken?

Mijn hele groep.

## Taak

### Welke rol had je?

Mijn rol was bedrijfsman, gericht plannen maken en dingen uitvoeren. Wat ik ook deed, ik voerde mijn taken uit en had ook de presentatie van sprint 2 structuur gegeven en geplanned.

### Wat wilde je bereiken?

Ik wilde mijn project goed maken en een goede sweer in ons team hebben.

### Wat werd er van je verwacht of wat verwachtte jezelf?

Van mij werd er verwacht dat ik mijn taken af zou krijgen zoals mijn eigen kamer en ik verwachtte zelf dat iedereen ook zijn taken zou afmaken.

## Actie

### Wat was je aanpak en waarom?

Mijn aanpak was goed communiceren over de story line en wat nog gedaan moest worden. Waardoor iedereen wist waar die mee bezig was. Mijn aanpak voor OOP was om te kijken hoe alles een beetje werkte kwa OOP in Typescript + wat mogelijk was, in vergelijking tot Java.

### Waarom heb je het zo aangepakt? (Dubbele vraag?)

Omdat dit mij de de beste manier leek te zijn. En het ging ook goed dus geen reden om een andere aanpak te hebben. 

## Resultaat

### Heeft het gewerkt?

Het heeft gewerkt, ik weet nu wat er mogelijk is kwa OOP in Typescript. Ook is de communicatie goed gegaan aangezien we op niveau hebben voor onze presentatie.

### Waarom heeft het gewerkt?

Omdat we goed gecommuniceerd en samengewerkt hebben.

## Reflectie

### Wat heb je er van geleerd?

Dat communicatie belangrijk is als je in dezelfde code base werkt en mijn hypothese dat alle programmeer talen wel ongeveer het zelfde zijn.

### Was je tevreden met de resultaten?

Ik ben zeer tevreden met de resultaten.

### Wat is de essentie van wat je geleerd hebt?

Hoe je samenwerkt in een team, en hoe je retrospectives schrijft.

## Transfer

### Hoe pas je dit toe en wat is de volgende stap?

Ik ga volgende keer weer goed communiceren en mijn taken afmaken. Ik wil nog meer groeien in Typescript en ook echt alles zelf kunnen schrijven / alleen begrijpen wat de code doet.

### Kun je dit ook toepassen in andere situaties?

Dit kan ik toepassen in situaties waarbij ik in teams werk.

### Wat zou je een volgende keer anders of beter doen?

Meer kijken naar vergelijkbare projecten voor inspiratie.
